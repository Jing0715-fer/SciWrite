     1→# SciWrite — AI Research Literature Writing Assistant · Worklog
     2→
     3→## Project Status (as of initial delivery)
     4→
     5→**Status: ✅ Functional MVP delivered and browser-verified.**
     6→
     7→SciWrite is a single-page Next.js 16 application that helps researchers:
     8→1. Query live scientific databases (RCSB PDB, UniProt, PubMed, NCBI Gene, NCBI BLAST).
     9→2. Save records as **data sources** or **references**.
    10→3. Let an AI draft **citation-backed scholarly paragraphs** from a topic + selected
    11→   sources/references + optional live web search.
    12→4. **Annotate** paragraph text (text selection → comment/type/severity) and run
    13→   **AI revision** (annotations-mode, instructions-mode, or polish-mode).
    14→5. **Compose** multiple paragraphs into a deeper research article with section
    15→   headings, preserved inline citations, and an aggregated references list.
    16→
    17→All database/AI calls run server-side via `z-ai-web-dev-sdk` (LLM, web_search,
    18→page_reader) and direct public APIs (NCBI E-utilities, UniProt REST, RCSB search).
    19→
    20→## Architecture
    21→
    22→- **Framework**: Next.js 16 (App Router) + TypeScript + Tailwind 4 + shadcn/ui (New York).
    23→- **DB**: Prisma + SQLite (`prisma/schema.prisma`). Models: Project, Paragraph,
    24→  Annotation, Reference, DataSource, Article, ArticleParagraph.
    25→- **State**: TanStack React Query for server state; React hooks for UI.
    26→- **Theme**: Emerald/teal academic palette (NO indigo/blue), Lora serif for prose,
    27→  Geist sans for UI, paper-texture surfaces, custom citation-marker + annotation
    28→  highlight styles in `globals.css`.
    29→- **Layout**: `h-screen flex flex-col` → sticky header + 3-panel resizable workspace
    30→  (Projects sidebar · Writing workspace · Databases+Knowledge panel) + sticky footer.
    31→
    32→### Key files
    33→- `src/lib/types.ts`, `src/lib/constants.ts` — shared types + presets (formats, scenarios, statuses).
    34→- `src/lib/ai.ts` — ZAI SDK wrapper (chat / webSearch / readPage).
    35→- `src/lib/databases.ts` — RCSB/UniProt/PubMed/NCBI/BLAST clients + router.
    36→- `src/lib/writing.ts` — prompt builders (write/revise/compose), citation context, word count.
    37→- `src/lib/api-client.ts` — typed fetch helpers for all endpoints.
    38→- `src/app/api/**` — REST routes (projects, paragraphs, annotations, references,
    39→  data-sources, articles, ai/write, ai/compose, databases, search, reader).
    40→- `src/components/sciwrite/**` — UI: projects-sidebar, database-query-panel,
    41→  topic-composer, paragraph-card, markdown-citations, knowledge-panel,
    42→  article-composer, theme-toggle, icon.
    43→- `src/app/page.tsx` — single route `/`, orchestrates the whole app.
    44→
    45→### Citation model
    46→Inline markers are written by the LLM as `[n]` (reference-list index) or
    47→`[SOURCE:ID]` (e.g. `[PMID:12345678]`, `[PDB:1A3N]`, `[UniProt:P04637]`).
    48→`MarkdownCitations` parses these into styled `<sup class="cite-marker">` spans and
    49→overlays annotation highlights by matching `selectedText` against the content.

    80→   the browser** (programmatic selection is fiddly with agent-browser). The
    81→   `SelectionToolbar` popover + `onMouseUp` handler are wired; manual click-test

    88→   `selectedText`; rare overlapping selections are deduped by severity priority.
    89→5. No persistence of "drafted but unsaved" AI output — the TopicComposer must be
    90→   saved explicitly.
