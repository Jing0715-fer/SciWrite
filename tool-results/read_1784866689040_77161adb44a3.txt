     3‚Üí     3‚Üí## Project Status (as of initial delivery)
     4‚Üí     4‚Üí
     5‚Üí     5‚Üí**Status: ‚úÖ Functional MVP delivered and browser-verified.**
     6‚Üí     6‚Üí
     7‚Üí     7‚ÜíSciWrite is a single-page Next.js 16 application that helps researchers:
     8‚Üí     8‚Üí1. Query live scientific databases (RCSB PDB, UniProt, PubMed, NCBI Gene, NCBI BLAST).
     9‚Üí     9‚Üí2. Save records as **data sources** or **references**.
    10‚Üí    10‚Üí3. Let an AI draft **citation-backed scholarly paragraphs** from a topic + selected
    11‚Üí    11‚Üí   sources/references + optional live web search.
    12‚Üí    12‚Üí4. **Annotate** paragraph text (text selection ‚Üí comment/type/severity) and run
    13‚Üí    13‚Üí   **AI revision** (annotations-mode, instructions-mode, or polish-mode).
    14‚Üí    14‚Üí5. **Compose** multiple paragraphs into a deeper research article with section
    15‚Üí    15‚Üí   headings, preserved inline citations, and an aggregated references list.
    16‚Üí    16‚Üí
    17‚Üí    17‚ÜíAll database/AI calls run server-side via `z-ai-web-dev-sdk` (LLM, web_search,
    18‚Üí    18‚Üípage_reader) and direct public APIs (NCBI E-utilities, UniProt REST, RCSB search).
    19‚Üí    19‚Üí
    20‚Üí    20‚Üí## Architecture
    21‚Üí    21‚Üí
    22‚Üí    22‚Üí- **Framework**: Next.js 16 (App Router) + TypeScript + Tailwind 4 + shadcn/ui (New York).
    23‚Üí    23‚Üí- **DB**: Prisma + SQLite (`prisma/schema.prisma`). Models: Project, Paragraph,
    24‚Üí    24‚Üí  Annotation, Reference, DataSource, Article, ArticleParagraph.
    25‚Üí    25‚Üí- **State**: TanStack React Query for server state; React hooks for UI.
    26‚Üí    26‚Üí- **Theme**: Emerald/teal academic palette (NO indigo/blue), Lora serif for prose,
    27‚Üí    27‚Üí  Geist sans for UI, paper-texture surfaces, custom citation-marker + annotation
    28‚Üí    28‚Üí  highlight styles in `globals.css`.
    29‚Üí    29‚Üí- **Layout**: `h-screen flex flex-col` ‚Üí sticky header + 3-panel resizable workspace
    30‚Üí    30‚Üí  (Projects sidebar ¬∑ Writing workspace ¬∑ Databases+Knowledge panel) + sticky footer.

    45‚Üí    45‚Üí### Citation model
    46‚Üí    46‚ÜíInline markers are written by the LLM as `[n]` (reference-list index) or
    47‚Üí    47‚Üí`[SOURCE:ID]` (e.g. `[PMID:12345678]`, `[PDB:1A3N]`, `[UniProt:P04637]`).
    48‚Üí    48‚Üí`MarkdownCitations` parses these into styled `<sup class="cite-marker">` spans and
    49‚Üí    49‚Üíoverlays annotation highlights by matching `selectedText` against the content.

    78‚Üí    78‚Üí### Known limitations
    79‚Üí    79‚Üí1. **Text-selection annotation is backend-verified but not yet clicked-through in
    80‚Üí    80‚Üí   the browser** (programmatic selection is fiddly with agent-browser). The
    81‚Üí    81‚Üí   `SelectionToolbar` popover + `onMouseUp` handler are wired; manual click-test
    82‚Üí    82‚Üí   recommended. Revise/annotate API path is fully verified.
    83‚Üí    83‚Üí2. **BLAST** uses NCBI's async RID-polling (up to ~60s). For very long queries it
    84‚Üí    84‚Üí   may time out; a "check back later" fallback message is shown.
    85‚Üí    85‚Üí3. **Web search enrichment** in AI Write uses `z-ai functions.invoke('web_search')`;
    86‚Üí    86‚Üí   results are passed as context but not yet auto-saved as references.
    87‚Üí    87‚Üí4. Annotation highlight matching is `indexOf`-based (case-sensitive) on
    88‚Üí    88‚Üí   `selectedText`; rare overlapping selections are deduped by severity priority.
    89‚Üí    89‚Üí5. No persistence of "drafted but unsaved" AI output ‚Äî the TopicComposer must be
    90‚Üí    90‚Üí   saved explicitly.
